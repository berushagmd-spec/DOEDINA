# 🟥DERD GENERATOR 95🟥

Рофл-бот с четырьмя отдельными генераторами персонажей. Каждый раз он создаёт
новое имя и случайный 24-битный HEX-цвет. Три ГМД-персонажа перекрашиваются, а
на фотографию Тимура накладывается Дерд выбранного цвета.

## Команды

- `/genderd` — случайный Дерд
- `/genrom` — случайный Ромикс Татар
- `/gennel` — случайный Дима Нелис
- `/gentim` — случайный Тимур Бабаев
- `/help` — показать подсказку

Команды автоматически появляются в меню Telegram и работают в личных чатах,
обычных группах, супергруппах и темах. В группе также поддерживается форма
`/genrom@ИмяБота`.

## Правила генерации

### Дерд

Каждое имя обязательно заканчивается на `-ерд`, `-рд` или `-ед`. Процедурное
пространство содержит примерно 202 миллиарда сырых сочетаний.

### Ромикс Татар

Первая часть всегда заканчивается на `-икс`: например, «Ромикс», «Громикс» или
«Квадрикс». Вместо «Татар» выбирается случайная национальность, народ или этнос
из большой встроенной базы. Вместе это даёт более 4,5 миллиарда сочетаний.

### Дима Нелис

Имя выбирается из международной базы, причём славянские имена выпадают с
вероятностью 82%. Вторая часть всегда заканчивается на `-ис`. Пространство
сочетаний превышает 3 миллиарда.

### Тимур Бабаев

Имя выбирается из международной мужской базы, включая «Тимур». Сгенерированная
фамилия обязательно заканчивается на `-ев`. Пространство сочетаний превышает
3 миллиарда. На лицо в случайном месте накладывается немного повёрнутый Дерд
случайного HEX-цвета, поэтому фотографии отличаются.

## Очередь

Изображения обрабатывает ровно один worker. Первая генерация запускается сразу,
вторая и последующие ждут в FIFO-очереди и получают сообщение со своей позицией.
По умолчанию допускается до 50 ожидающих запросов; лимит задаётся переменной
`MAX_GENERATION_QUEUE`.

## Лог-канал и номера

Каждая успешно отправленная пользователю генерация дублируется в
`@GMDGenerator`. Бот должен быть администратором этого канала с правом публикации.
Подпись лога содержит строго две строки:

```text
ИМЯ
#95 генерация
```

Номер хранится в `generation_counter.txt` и не сбрасывается при обычном
перезапуске. На хостинге с временной файловой системой укажи через
`GENERATION_COUNTER_FILE` путь на подключённом постоянном диске.

## Быстрый запуск

1. Создай бота через `@BotFather` и скопируй токен
2. Переименуй `.env.example` в `.env`
3. Вставь токен в строку `BOT_TOKEN=...`
4. Добавь бота администратором канала `@GMDGenerator`
5. Установи зависимости и запусти:

```bash
python -m pip install -r requirements.txt
python bot.py
```

Токен нельзя выкладывать на GitHub. Файл `.env` уже добавлен в `.gitignore`.
На хостинге создай секретную переменную окружения `BOT_TOKEN`, а командой запуска
укажи `python bot.py`. Для worker-хостингов также приложен `Procfile`.

## Добавление в группу

Добавь бота как обычного участника. Администраторские права ему не нужны. Команды
видны при включённом Privacy Mode. Если в группе несколько ботов, Telegram может
добавить к команде имя бота — этот вариант тоже обрабатывается.

## Проверка без Telegram

```bash
python self_test.py
```

Тесты проверяют обязательные окончания всех четырёх генераторов, базы имён и
этносов, новые команды, меню Telegram, Дерда на фотографии, последовательную
очередь, постоянный счётчик и точный формат записи в лог-канале.
